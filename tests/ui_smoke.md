# UI Smoke Tests (Ручное тестирование)

## 1. Регистрация и Авторизация
-  Регистрация: Переходим на страницу регистрации, вводим валидные данные. Ожидается успешное создание пользователя и переход в рабочую область.
-  Вход: Выходим из аккаунта и логинимся заново с созданным логином/паролем. Ожидается успешный вход.
-  Валидация: Попытка зарегистрироваться с уже существующим именем показывает понятную ошибку.

## 2. Настройки (Settings)
-  Открытие: При переходе в настройки загружаются дефолтные значения (светлая тема).
-  Обновление: Меняем тему со светлой на тёмную. Страница меняет внешний вид. После перезагрузки тема сохраняется.
-  Неизвестное поле: Отправка несуществующего параметра не ломает настройки.

## 3. Доски (Boards)
-  Создание доски: Нажимаем «Создать доску», вводим название. Доска появляется в списке.
-  Пустое имя: Попытка создать доску с именем из пробелов блокируется с ошибкой.
-  Переименование: Изменяем название доски — обновлённое название сразу отображается.
-  Удаление: Удаляем доску — она исчезает из списка.

## 4. Задачи (Tasks)
-  Создание: Создаём задачу внутри выбранной доски с названием, статусом, приоритетом и описанием. Задача появляется в нужной колонке.
-  Редактирование: Меняем приоритет и описание — изменения сохраняются.
-  Удаление: Удаляем задачу — она исчезает с доски.

## 5. Drag and Drop (Перетаскивание)
-  Смена статуса: Перетаскиваем карточку из «To Do» в «In Process». После перезагрузки статус сохраняется.

## 6. Фильтрация (если реализована на фронтенде)
-  По статусу: Выбираем показ только задач со статусом «Done» — остальные скрываются.
-  По приоритету: Включаем фильтр «High priority» — остаются только важные задачи.
-  Комбинирование: Несколько фильтров одновременно — условия пересекаются корректно.

## 7. Изоляция пользователей
-  Пользователь A создаёт доску. Пользователь B не видит эту доску в своём списке.
-  Прямой переход по ссылке на чужую доску показывает 404 или редирект.
